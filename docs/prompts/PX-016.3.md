# PX-016.3 - Post-Launch Content Hooks
**Targets:** RM-016, RM-019  
**Status:** Planned  
**Owner:** Solo Dev

## Scope
Prepare the telemetry toggles, sandbox hooks, and dashboard alerts needed to introduce LiveOps events without code changes.

## Objectives
1. Add runtime toggles for events, layout boosts, and comfort experiments.
2. Record toggle state in telemetry and nightly dashboards.
3. Provide JSON or TSV configuration that designers can edit safely.
4. Document rollout and rollback procedures.

## Dependencies
- Event loop implementation (PX-016.2).
- Layout adjacency integration (PX-019.1).
- LiveOps runbook (PX-017.3).

## Deliverables
- Toggle manager scripts and configuration files.
- Dashboard updates highlighting active toggles.
- Documentation in `docs/ops/Release_Playbook.md` and `docs/qa/Metrics_Dashboard_Spec.md`.
- Example rollout checklist appended to `docs/ops/Release_Playbook.md`.

## Acceptance Criteria
1. Toggling a LiveOps configuration updates the game at runtime and logs the change.
2. Dashboard reflects active toggles within one report cycle.
3. Rollback instructions tested and documented.
4. Telemetry alerts trigger when toggles drive metrics outside guardrails.
